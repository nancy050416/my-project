# Agent 开放式能力测评提交说明

## 作品名称

ThinkLoop - 企业知识库与项目资料智能 Agent

## 高频真实场景

企业或项目团队每天都会反复处理大量内部资料问题，例如制度查询、接口说明、项目排障、会议纪要追溯和标准流程确认。普通大模型容易出现两个问题：一是无法稳定引用私有知识，二是长耗时任务没有过程反馈，用户不知道系统是否正在检索、调用工具或生成答案。

ThinkLoop 选择这个场景作为 Agent 作品基础：让用户上传或维护 Markdown 知识文档，系统将文档切分为知识片段并写入向量库；当用户提问时，Agent 根据上下文判断是否需要检索知识库、调用工具或继续多轮推理，并通过 SSE 把执行过程实时推送给前端。

## 核心工作流

```text
用户提问
-> 创建 Agent 运行实例
-> 加载历史上下文
-> think(): 判断下一步动作
-> 可选执行 RAG 检索或工具调用
-> 写入工具结果和检索结果
-> 继续多轮思考
-> 输出结构化答案并结束运行
```

为了避免 Agent 无限循环，系统设置最大执行步数；为了提升可观测性，每一轮思考、工具调用、异常和结束状态都会通过 SSE 返回前端。

## 系统模块

| 模块 | 作用 |
| --- | --- |
| Agent Loop | 负责多轮思考、动作选择、工具执行和终止判断 |
| Tool Calling | 手动接管工具调用生命周期，便于记录、重试、审计和扩展权限 |
| RAG 知识库 | 解析 Markdown 文档，生成 chunk，写入 PostgreSQL + pgvector |
| 多模型注册表 | 支持 DeepSeek / ZhipuAI 等不同 ChatModel 切换 |
| SSE 实时反馈 | 将 Agent 执行过程推送到前端，改善长任务体验 |
| React 前端 | 提供对话、知识库、会话和执行状态展示界面 |

## 创新与亮点

- 不是一次性问答，而是可多轮推进的 Agent Loop。
- 关闭默认自动 Tool Calling，由后端手动掌控工具调用过程，便于做权限、日志和可观测性。
- 将私有知识放入 RAG，而不是全部写进 Prompt，降低幻觉和上下文浪费。
- 使用 PostgreSQL + pgvector 统一管理文档元数据、chunk 和 embedding，减少跨系统同步复杂度。
- 用 SSE 展示实时执行过程，让用户能看到 Agent 正在思考、检索、调用工具或结束。
- 前后端分离，后续可继续扩展混合检索、rerank、工具权限、评测集和 Prompt 版本管理。

## 后续可优化方向

- 增加 RAG 评测集，系统评估 Top-K 召回率和答案引用准确率。
- 引入 BM25 + 向量检索的混合召回，并加入 rerank。
- 为工具调用增加权限控制、失败重试和审计日志。
- 增加 Prompt 版本管理，记录不同策略对 Agent 效果的影响。
- 将 Agent 运行轨迹结构化存储，用于复盘、调试和效果分析。
